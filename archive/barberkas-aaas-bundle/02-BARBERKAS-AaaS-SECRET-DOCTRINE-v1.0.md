| Incumbent | Kenapa tidak masuk segment kita |
|---|---|
| **Moka POS** (Indonesia) | Generic POS, butuh "vertical specialist" PM yang gak ada di tim. Barbershop = 0.5% TAM mereka, ROI gak masuk. |

**Conclusion**: Niche too small + too localized untuk semua incumbent. Window kita 12-18 bulan minimum.

## §3. Anti-Patterns (Things We Will NOT Do)

### 3.1 Will NOT — Build native mobile app

**Why**: App store review delay, install friction, update friction, Apple/Google policy risk.
**Instead**: Progressive Web App (PWA) di `<tenant>.barberkas.sparkmind.web.id`. Add to home screen = app-like.

### 3.2 Will NOT — Build hardware bundle

**Why**: CapEx kill margin, logistics nightmare, repair ops.
**Instead**: BYOD (capster pakai HP/tablet sendiri). Bluetooth thermal printer opsional via partner.

### 3.3 Will NOT — Accept VC funding pre-Rp 50jt MRR

**Why**: Sovereign play = no dilution. Niche TAM tidak butuh VC scale.
**Instead**: Bootstrap ke Rp 50jt MRR. Kalau scaling internasional baru consider strategic angel.

### 3.4 Will NOT — Add salon perempuan / spa

**Why**: Different industry, different agent training, different customer behavior. Dilusi focus.
**Instead**: Lock barbershop until 1.000 tenant. Adjacent vertical = brand baru (SparkMind sub).

### 3.5 Will NOT — Price-compete with Moka/Pawoon

**Why**: Race to bottom. Mereka punya scale economics.
**Instead**: Premium-anchor di Rp 299K-499K/bln untuk AaaS tier. Justify via measurable ROI.

### 3.6 Will NOT — Promise "10x revenue increase"

**Why**: Unproveable, brand-damaging kalau gagal, attracts wrong customer (get-rich-quick).
**Instead**: Promise "save 10 jam/bulan ops time" + "1-3 booking tambahan/minggu". Demonstrable, conservative.

### 3.7 Will NOT — Use unofficial WA API at scale

**Why**: Fonnte = unofficial. Bisa ban device. Risk = single point of failure.
**Mitigation**: Multi-device pool, fallback ke WA Business API resmi (Meta) di tier Enterprise, customer education up-front.

### 3.8 Will NOT — Outsource customer support to chatbot 100%

**Why**: Capster butuh trust, bukan auto-reply.
**Instead**: Tier-1 chatbot, tier-2 founder/team WA langsung dalam 4 jam.

## §6. Failure Modes (Pre-Mortem)

Skenario kita FAIL di D+180 (≤8 paying AaaS):

1. **Demo-day phenomenon**: Banyak interest, sedikit convert. Root cause = pricing > demonstrated value. Fix = lower tier + ROI proof point.
2. **WA Fonnte mass-ban**: Device kena ban → service down → churn. Fix = multi-provider redundancy (Wablas, WAPI backup).
3. **Agent quality drift**: Groq llama-3.3-70b deprecated atau pricing jump 10x. Fix = multi-LLM abstraction layer dari Sprint 0.
4. **Solo founder burnout**: Gyss capster + dev + sales = bottleneck. Fix = Adik co-founder activate by Sprint 2, recruit junior helper Sprint 3.
5. **Niche too small**: 5.000 barbershop overestimate. Real addressable maybe 1.500. Fix = expand ke salon barber-style (mixed) tier-3 city.
6. **MoR Duitku revoke**: Oasis BI Pro KYC issue. Fix = backup PG Midtrans/Xendit pre-integration Sprint 1.

**Pre-mortem result**: All failure modes have mitigation. Risk = manageable, bukan fatal.
