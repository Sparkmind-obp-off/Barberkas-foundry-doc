## §4. Cross-Brand Customer Journey (Pollination)

### 4.1 Eligible Cross-Sell Flows

**BarberKas merchant → KuratorKas**
- Trigger: Merchant has fashion/apparel side business (detected via WA convo or merchant profile)
- Pitch: "Lihat KuratorKas? Sama-sama AI curator tapi untuk produk fashion kamu."
- Commission: 10% to BarberKas (origin brand)

**BarberKas merchant → PaceLokal** (Banyumas-aware)
- Trigger: Merchant is club running active (Banyumas community overlap)
- Pitch: "Klub lari kamu juga butuh tools? Coba PaceLokal."
- Commission: 10% to BarberKas

**KuratorKas / PaceLokal → BarberKas**
- Reciprocal flow when fashion or running merchant has barbershop side biz
- Commission: 10% to origin brand

### 4.2 Doctrine Lock
- Cross-sell ONLY if merchant pays current brand 3+ months (proven retention)
- Cross-sell ONLY 1 offer per quarter per merchant (anti-spam)
- Founder personal WA, never automated bulk

## §7. Cross-Brand Risk Containment

### 7.1 Failure Isolation
- D1 separate database per brand → one brand corrupt data ≠ all corrupt
- Workers separate per brand → one brand crash ≠ all crash
- R2 prefix separation → no leak
- Fonnte device per tenant (and brand) → ban one ≠ ban all

### 7.2 Reputational Spillover
- Single founder name across brands = good or bad PR spreads
- Doctrine: "First brand to gain bad PR = pause all promotion across brands 30 days, investigate, recover"

### 7.3 Resource Allocation Discipline
- P0 (BarberKas) = 80% Gyss time
- P1 (KuratorKas + PaceLokal) = 15% (landing maintenance)
- P2 (Nurani.OS) = 5% (research only)
- Quarterly rebalance based on revenue performance
